Writing Assignment One

A well-designed software that I have used is Microsoft’s Visual Studio. This is due it being easy to use while either directly having capabilities or the ability to use extensions or other means to do anything you need to do with it. Proper software design is important for many reasons. It increases the speed of development and reduces errors by making code readable and easy to understand. It can help prevent errors and compromises that will cause the end user to have to fight with the program instead of just using it. This worsens the user experience, leading to poor reviews, less previous users coming back, less new customers, and less money for developer. In short, proper software design will allow you to produce better programs faster, have more and happier users, and more money if you are selling your program.

The requirement’s gathering stage is important as it tells you what your client wants for the project. Preforming the requirements gathering stage properly allows you to understand what the client wants, plan out what needs to be done and allows you to tell the client if they are asking for too much. Not preforming this step leads to you potentially repeating all the other steps and can increase the time it takes to preform all the following steps in the process. For example, not preforming this step can cause the client to keep asking for more features, leading to convoluted design of the software, increasing the time it takes to develop the software and making it a nightmare for new people to work on and maintain.
